# Medfarms · Michigan Multi-Brand Sales Dashboard

Live, self-updating dashboard for the **Medfarms – 100 Shafer Processing**
LeafLink account (seller id `9105`), showing **every brand** on the account —
not just Chill Medicated. Data is scraped from the LeafLink Marketplace V2 API
by a GitHub Action and rendered by a single-file React/Recharts dashboard on
GitHub Pages.

```
medfarms-mi-multibrand/
├── scraper.py                  ← pulls ALL brands from LeafLink → sales_data.json
├── dashboard.html              ← the multi-brand dashboard UI (brand tabs)
├── sales_data.json             ← generated by the scraper (committed, do NOT gitignore)
├── logo.png                    ← Chill / Medfarms mark shown in the header
├── requirements.txt
├── .gitignore
└── .github/workflows/scrape.yml
```

This is the same concept as the other dashboards (Missouri Apex, the
Chill-only Michigan board), with two differences: the scraper's brand filter is
**off** so all brands flow through, and the dashboard adds **brand tabs** plus
**account routing** (exclude / internal-store / wholesale).

---

## What the dashboard shows

- **Brand tabs** across the top — `All Brands` plus one tab per brand, each with
  its wholesale revenue. Tabs are generated from the data, so new brands appear
  automatically.
- **Wholesale (external)** section per brand: KPIs, revenue trend, top
  products/buyers, sales by rep, category mix, by-state, payment status, a
  monthly breakdown, and an invoice table (one row per order).
- **Internal Stores** section (own segment, kept out of wholesale totals): KPIs,
  per-store table, and internal invoices.
- **All Brands** tab also shows a brand-comparison chart (wholesale vs internal).
- **Account Routing** panel (collapsible, near the top) listing exactly which
  accounts landed in each bucket — use it to verify the routing.

## Account routing — edit in `dashboard.html`

Near the top of `dashboard.html` there are two lists. Matching is
**"buyer name starts with," case-insensitive**, so a base entry like
`Dispo - Romeo` also catches `Dispo - Romeo MED`.

```js
const EXCLUDE_ACCOUNTS = [        // removed from EVERY total
  "Chill CBD Ecommerce",
  "Dispo Ace Production",
  "GNO",                          // matches "GNO Enterprises - Backpack Boyz - Monroe"
];
const INTERNAL_STORES = [         // pulled out of wholesale, shown in their own box
  "Dispo - Bay City North",
  "Dispo - Bay City South",
  "Dispo - Battle Creek (Fire Creek)",   // the "Dispo - Fire Creek Battle Creek" store
  "Dispo - Hazel Park",
  "Dispo - Romeo",
  "Dispo - Whitmore Lake",
];
```

Order of precedence: **exclude → internal → otherwise wholesale**. The routing is
done entirely in the dashboard, so the scraper keeps pulling everything and you
can reclassify accounts without re-scraping.

## First-time setup

### 1. Create the repo and add these files
Upload everything in this folder. Keep `sales_data.json` **tracked** — if it's
in `.gitignore`, the Action can't commit fresh data.

### 2. Add the API key as a secret
Settings → Secrets and variables → Actions → **New repository secret**:

| Secret | Value |
|---|---|
| `LEAFLINK_API_KEY` | your LeafLink App token (scoped to the Medfarms company) |

The scraper sends it as `Authorization: App <key>` against
`https://www.leaflink.com/api/v2/`.

### 3. Turn on GitHub Pages
Settings → Pages → Deploy from branch → `main` / root. Your dashboard will be at
`https://<user>.github.io/<repo>/dashboard.html`.

### 4. Populate data
Run the **Scrape LeafLink Sales (All Brands)** workflow once (Actions tab →
Run workflow), or push to `main`. It writes `sales_data.json` with every brand.

## Keeping data fresh (cron-job.org)

Trigger the workflow on a schedule via cron-job.org hitting GitHub's
`workflow_dispatch` API — same pattern as the other repos.

- **POST** `https://api.github.com/repos/<user>/<repo>/actions/workflows/scrape.yml/dispatches`
- Headers: `Authorization: Bearer <PAT>`, `Accept: application/vnd.github+json`
- Body: `{"ref":"main"}`

The PAT must be **fine-grained** with **Actions: Read and write** on this repo
(Actions, not Workflows). Don't also enable the `schedule:` block in the
workflow — pick one scheduler to avoid overlapping runs.

## Local run

```bash
pip install -r requirements.txt
export LEAFLINK_API_KEY="...your app token..."
export LEAFLINK_BRAND=""          # all brands
python scraper.py                 # writes sales_data.json
python -m http.server 8000        # then open http://localhost:8000/dashboard.html
```

## Notes

- **Revenue** is net per line: each order's lines are scaled so they sum to the
  LeafLink order total (distributes order-level discount/tax/shipping), so the
  dashboard ties out to LeafLink's order totals.
- **Brands** come straight from each line's product, so the brand filter being
  empty is all it takes to get the full picture.
- **Internal stores** are excluded from wholesale totals by design. If you'd
  rather count them in each brand's wholesale total *and* keep the separate box,
  that's a small dashboard change — just ask.
